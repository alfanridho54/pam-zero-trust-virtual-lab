## Do Not Screenshot

| File / area | Alasan tidak direkomendasikan |
|---|---|
| `app/Console/Commands/TerminalWebSocketServer.php` bagian handshake/frame WebSocket mentah, sekitar baris 75-109 dan 369-453 | Terlalu low-level untuk tesis implementasi PAM; fokusnya protokol WebSocket, bukan konsep PTY, audit, atau kontrol akses. Ambil hanya `authenticate()`, `handlePayload()`, `writePty()`, atau `pumpPtyClient()`. |
| `app/Services/ProxmoxService.php` helper umum seperti `request()`, `apiRequest()`, `normalizeResponse()`, dan `safeJson()` | Berguna secara teknis, tetapi terlalu generik. Lebih baik screenshot `cloneStudentVm()`, `getTaskStatus()`, atau lifecycle action yang langsung terkait provisioning. |
| `resources/views/...` untuk Bab 4.2 | View Blade menunjukkan UI, sedangkan permintaan ini fokus pada screenshot kode implementasi. Simpan screenshot UI untuk 4.3 jika diperlukan sebagai bukti hasil. |
| `routes/web.php` dan `routes/api.php` | Route hanya menghubungkan endpoint ke controller; kurang kuat sebagai bukti implementasi dibanding controller/service. |
| `app/Models/Vm.php` seluruh file | File ini panjang dan banyak helper domain. Ambil hanya blok kecil seperti `proxmoxVmid()`, `isProvisionedStudentVm()`, atau SSH metadata jika benar-benar dibutuhkan. |
| `app/Http/Controllers/DashboardController.php` seluruh file | Terlalu panjang dan campur banyak fitur. Untuk audit gunakan hanya query timeline atau method `audit()`; untuk provisioning admin/bulk jangan dicampur dengan self-service mahasiswa. |
| `app/Services/SshCommandService.php` | Lebih cocok untuk terminal command non-PTY. Jangan dijadikan screenshot utama subbab PTY kecuali tesis juga membahas fallback command execution. |
| `database/seeders/*` dan migration unrelated | Seeder bukan implementasi runtime. Gunakan migration audit log hanya jika ingin menunjukkan struktur tabel. |
| `.env` atau konfigurasi rahasia | Jangan screenshot karena berpotensi mengekspos token, host internal, password, atau secret Cloudflare/Proxmox. |
